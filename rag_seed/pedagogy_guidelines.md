# Pedagogical guidelines (seed corpus)

This seed corpus is intentionally short and generic so it can be indexed locally.
You can extend/replace it with your institution's assessment policies.

## Clear learning outcomes & alignment
- Define the **intended learning outcome (ILO)** in observable terms (e.g., "analyze", "design", "evaluate").
- Ensure **construct alignment**: question prompt, expected answer, and rubric should measure the same skill.

## Cognitive level (Bloom-style)
- **Remember / Understand**: definitions, explanation in own words, simple examples.
- **Apply**: use a method on a new instance, compute, implement, execute a procedure.
- **Analyze**: compare/contrast, diagnose, identify assumptions, break down into parts.
- **Evaluate**: justify a choice with criteria, critique a solution, weigh trade-offs.
- **Create**: propose a novel design, synthesize concepts, develop an original solution.

## Item writing basics
- Use unambiguous language and define constraints.
- Avoid "trick" wording and unnecessary complexity unrelated to the target skill.
- If multiple parts exist, label them and allocate marks per part.
- Make the expected format explicit (steps, final answer only, diagram, pseudocode, etc.).

## Difficulty vs discrimination (conceptual)
- Difficulty relates to how many students answer correctly.
- Discrimination relates to whether the item differentiates high vs low performers.
- Extremely easy/hard items often discriminate poorly; moderate difficulty can improve discrimination.

## Rubrics & scoring
- Provide **criteria-based** rubrics: list dimensions (correctness, reasoning, methodology, clarity).
- Describe **partial credit** rules for common error patterns.
- Keep rubrics consistent across similar items.

## Fairness & accessibility
- Avoid cultural/linguistic bias and irrelevant prerequisites.
- Provide necessary context/data inside the question.
- Consider accommodations: readable formatting, clear notation, alternatives for visual content.

## Feedback & learning gaps
- Map common wrong answers to likely misconceptions.
- Provide targeted feedback suggestions per misconception.

## Responsible use / academic integrity
- Make expectations explicit about tool use (e.g., calculator, internet, LLM tools).
- Design items that reward reasoning and process, not only final answers.
